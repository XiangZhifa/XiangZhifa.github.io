JavaScript 浮点数精度

发表于 2022-07-20 | 分类于 Javascript

数字类型

四种表示浮点数值的方式：单精确度（32位）、双精确度（64位）、延伸单精确度、与延伸双精确度。

像 ECMAScript 采用的就是双精确度，也就是说，会用 64 位来储存一个浮点数。

为什么 0.1+0.2 !== 0.3 为 true

0.1 和 0.2 转换为二进制，是循环数。对于这种情况，Js采取的是截取超出64位的部分，并且进1，进行存储。

所以 0.1 和 0.2 在存储时，发生了两次经度丢失。

最终导致了 0.1+0.2 !== 0.3 为 true。
