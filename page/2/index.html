Nginx 浏览器缓存相关

发表于 2022-04-06 | 分类于 Nginx

浏览器缓存执行流程

强缓存：不会发送请求到服务端，直接读取缓存

弱缓存：发送请求到服务端，服务端通过ETag和Last-Modified判断缓存是否失效，失效则返回新资源(code 200)，不失效则返回(code 304)直接读取缓存

expires指令

  1. 语法：expire time | epoch | max | off
  2. 位置：http server location

Nginx Gzip静态资源压缩

发表于 2022-04-06 | 分类于 Nginx

gzip指令

gzip指令由 ngx_http_gzip_module 模块支撑，ngx_http_gzip_module在Nginx安装时默认内置，所以可以直接使用gzip指令。

  1. 语法：gzip on | off
  2. 所属块：http server location

gzip_types指令

gzip_types指令会根据配置的MIME-type，选择性的对返回结果进行压缩。

  1. 语法：gzip_types MIME-type
  2. 所属块：http server location

gzip_comp_level指令

gzip_comp_level指令用于配置压缩级别，级别1-9。

级别1：表示压缩程度最低，耗时最短。

级别9：表示压缩程度最高，耗时最长。

  1. 语法：gzip_comp_level level
  2. 所属块：http server location

gzip_vary指令

gzip_vary指令用于设置响应头是否携带 “Vary:Accept-Encoding”，主要是为了告诉接收方数据已经过Gzip压缩处理。

  1. 语法：gzip_vary on | off
  2. 所属块：http server location

gzip_buffers指令

gzip_buffers指令用于指定处理压缩的缓冲区的数量和大小。

  1. 语法：gzip_buffers number buffers (number配置缓冲区数量 size配置缓冲区大小)
  2. 所属块：http server location

gzip_disable指令

gzip_disable指令用于配置针对部分浏览器，选择性关闭Gzip功能。

  1. 语法：gzip_disable regex …
  2. 所属块：http server location

gzip_http_version指令

gzip_http_version指令用于配置使用Gzip的http最低版本。

  1. 语法：gzip_http_version 1.0 | 1.1
  2. 所属块：http server location

gzip_min_length指令

gzip_min_length指令用于根据传输数据的大小，选择性开发Gzip压缩。(必须大于配置值，才会进行压缩。)

  1. 语法：gzip_min_length length
  2. 所属块：http server location

gzip_proxied指令

gzip_proxied指令设置是否对服务端返回的结果进行Gzip压缩。

  1. 语法：gzip_proxied off | expired | no-cache | no-store | private | no_last_modified | no_etag | auth | any
  2. 所属块：http server location

off: 关闭对返回结果的压缩

expired: 如果header包含”Expires”信息，启用压缩

no-cache: 如果header包含”Cache-control:no-cache”，启用压缩

no-store: 如果header包含”Cache-control:no-store”，启用压缩

private: 如果header包含”Cache-control:private”，启用压缩

no_last_modified: 如果header不包含”Last-Modified”，启用压缩

no_etag: 如果header不包含”ETag”，启用压缩

auth: 如果header包含”Authorization”，启用压缩

any: 无条件启用压缩

Nginx 静态资源优化配置

发表于 2022-04-06 | 分类于 Nginx

sendfile指令

sendfile指令用来开启高效的文件传输模式

  1. 语法：sendfile on | off
  2. 所属块：http server location …

tcp_nopush指令

tcp_nopush指令必需在sendfile打开的情况下才会生效。

tcp_nopush生效的情况下，会先将需要返回的数据存入缓存区，等缓存区存满后，再发送。

  1. 语法：tcp_nopush on | off
  2. 所属块：http server location

tcp_nodelay指令

tcp_nodelay指令必需在keepalive连接开启的情况下生效。

tcp_nodelay生效的情况下，会实时返回数据。

  1. 语法：tcp_nodelay on | off
  2. 所属块：http server location

Nginx root和alias指令

发表于 2022-03-23 | 分类于 Nginx

root指令

  1. 语法：root path
  2. 所属块：http，server，location

alias指令

  1. 语法：alias path
  2. 所属块：location

二者区别

  1. root转发的结果：root路径+location路径
  2. alias转发的结果：alias路径直接替换location路径
  3. 如果location以 / 结尾，alias也必须以 / 结尾，而root则没有这个要求
  4. root是定义最上层的目录含义，alias是定义目录的别名

Nginx location指令配置

发表于 2022-03-23 | 分类于 Nginx

location指令

  1. 语法：location [ = 、~ 、 ~* 、 ^~ 、 @]url (共六种匹配方式)
  2. 所属块：server , location

location 无符号 匹配

无符号：宽泛匹配，例如 /test123 依然能够访问 /test

location = 匹配

=：精确匹配，用于不含正则表达式的url前，例如 必须是/test 才能访问 /test

location ~ 和 ~* 匹配

~：表示url中包含正则表达式，且区分大小写

~*：表示url中包含正则表达式，且不区分大小写

^~匹配

^~：用于不含正则表达式的url前，效果与无符号一致，唯一不同的是，如果当前模式匹配，会停止继续向后搜索其他模式
