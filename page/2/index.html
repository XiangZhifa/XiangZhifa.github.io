Nginx root和alias指令

发表于 2022-03-23 | 分类于 Nginx

root指令

  1. 语法：root path
  2. 所属块：http，server，location

alias指令

  1. 语法：alias path
  2. 所属块：location

二者区别

  1. root转发的结果：root路径+location路径
  2. alias转发的结果：alias路径直接替换location路径
  3. 如果location以 / 结尾，alias也必须以 / 结尾，而root则没有这个要求
  4. root是定义最上层的目录含义，alias是定义目录的别名

Nginx location指令配置

发表于 2022-03-23 | 分类于 Nginx

location指令

  1. 语法：location [ = 、~ 、 ~* 、 ^~ 、 @]url (共六种匹配方式)
  2. 所属块：server , location

location 无符号 匹配

无符号：宽泛匹配，例如 /test123 依然能够访问 /test

location = 匹配

=：精确匹配，用于不含正则表达式的url前，例如 必须是/test 才能访问 /test

location ~ 和 ~* 匹配

~：表示url中包含正则表达式，且区分大小写

~*：表示url中包含正则表达式，且不区分大小写

^~匹配

^~：用于不含正则表达式的url前，效果与无符号一致，唯一不同的是，如果当前模式匹配，会停止继续向后搜索其他模式

Nginx http配置

发表于 2022-03-20 | 分类于 Nginx

http块

定义MIME-Type

MIME-Type用来区分Http请求的资源类型

在Nginx配置文件中，默认有如下配置

default_type指令

  1. 语法：default_type MIME-Type
  2. 默认值：default_type text/plain
  3. 位置：http块、server块、location块

定义服务日志

Nginx中日志类型分为，access.log、error.log

access.log：用来记录用户所有的访问请求

error.log：用来记录Nginx本身运行时的错误信息，不会记录用户的访问请求

Nginx支持对日志的格式、大小、输出等进行设置，需要用到两个指令 access_log 和 log_format

access_log指令

  1. 语法：access_log path name
  2. 默认值：access_log logs/access.log combined
  3. 位置：http块、server块、location块

log_format指令

  1. 语法：log_format name formatRule
  2. 默认值： log_format combined
  3. 位置：http块

send_file指令

send_file 设置Nginx是否使用sendfile()函数传输文件，该属性可以大大提高Nginx处理静态资源的性能

  1. 语法：send_file on|off
  2. 默认值：send_file off
  3. 位置：http块、server块、location块

keepalive_timeout指令

keepalive_timeout 设置长连接超时时间

  1. 语法：keepalive_timeout time
  2. 默认值：keepalive_timeout 75s
  3. 位置：http块、server块、location块

keepalive_requests指令

keepalive_requests 设置一个keepalive连接使用的次数

  1. 语法：keepalive_requests num
  2. 默认值：keepalive_requests 100
  3. 位置：http块、server块、location块

Nginx 全局配置 其他指令

发表于 2022-03-20 | 分类于 Nginx

全局块

daemon指令

daemon 设定nginx是否以守护进程的方式启动 (守护进程独立于控制终端，不会随控制终端的关闭而停止)

  1. 语法：daemon on|off
  2. 默认值：on

pid指令

pid 用来配置Nginx当前master进程的进程号id存储的文件路径

  1. 语法：pid filePath
  2. 默认值: /var/run/nginx.pid

error_log指令

error_log 用来配置Nginx错误日志的存放路径

  1. 语法：error_log filePath errorLevel

  2. 默认值：error_log /var/log/nginx/error.log notice;

  3. 错误级别：

     debug-调试消息

 info -信息性消息
 
 notice -公告
 
 warn -警告
 
 error -处理请求时出错
 
 crit -关键问题。需要立即采取行动
 
 alert -警报。必须立即采取行动
 
 emerg - 紧急情况。系统处于无法使用的状态

include指令

include 用来引入其他配置文件

  1. 语法：include filePath
  2. 配置位置：任意

Nginx events配置

发表于 2022-03-20 | 分类于 Nginx

events块

accept_mutex指令

accept_mutex 用来设置Nginx网络连接序列化

  1. 语法：accept_mutex on|off
  2. 默认值：accept_mutex on

multi_accept指令

multi_accept 用来设置是否允许worker进程同时接收多个网络连接

  1. 语法：multi_accept on|off
  2. 默认值：multi_accept off

worker_connections指令

worker_connections 用来配置单个worker进程最大连接数

  1. 语法：worker_connections num;
  2. 默认值：worker_connections 512;

use指令

use 设置Nginx使用哪种事件驱动来处理网络消息

  1. 语法：use method
  2. 默认值：根据不同操作系统而定 (CentOs可选值有select/poll/epoll/kqueue等)

Nginx 全局配置 work process

发表于 2022-03-20 | 分类于 Nginx

全局块

master_process指令

master_process 用来指定是否开启worker进程

  1. 语法：master_process on|off;
  2. 默认值：master_process on;

案例

worker_process指令

worker_process用来配置Nginx运行的worker进程的数量，worker进程的数量越大，能处理的并发数越大。受服务器自身性能限制，建议与服务器CPU内核数保持一致。

  1. 语法：master_process num|auto;
  2. 默认值：master_process auto;
